# Проверка конфликтов с Vercel

## Возможные проблемы

Если у вас был деплой на Vercel, он может конфликтовать с деплоем на REG.Ru:

### 1. DNS указывает на Vercel вместо REG.Ru

**Как проверить:**
1. Откройте https://dnschecker.org/
2. Введите домен `takto.space`
3. Проверьте A-запись:
   - Должна указывать на `31.31.196.49` (REG.Ru)
   - Если указывает на IP Vercel (например, `76.76.21.21` или другой) - это проблема

**Решение:**
1. В панели REG.Ru перейдите в **"Домены"** → **"Управление DNS"**
2. Убедитесь, что A-запись для `@` указывает на `31.31.196.49`
3. Если нужно, измените A-запись

### 2. Автоматический деплой на Vercel через GitHub

Если у вас подключен автоматический деплой на Vercel:
- Каждый `git push` может автоматически деплоить на Vercel
- Это может создавать конфликты

**Как проверить:**
1. Зайдите на https://vercel.com
2. Войдите в аккаунт
3. Проверьте, есть ли проект с доменом `takto.space`

**Решение:**
1. В Vercel отключите автоматический деплой для этого проекта
2. Или удалите проект из Vercel, если он больше не нужен
3. Или измените домен в Vercel на другой (например, `takto-vercel.vercel.app`)

### 3. Кэш Vercel CDN

Если раньше сайт был на Vercel:
- CDN Vercel может кэшировать старую версию
- Это может создавать проблемы с доступностью

**Решение:**
- Подождите 5-15 минут (время жизни кэша)
- Или очистите кэш в настройках Vercel (если проект еще существует)

## Текущая ситуация

**Сейчас сайт деплоится на REG.Ru через FTP:**
- ✅ Файлы загружаются в `/public_html` на сервере `31.31.196.49`
- ✅ Используется скрипт `deploy-ftp.js`
- ✅ Команда: `npm run deploy`

**Vercel НЕ используется:**
- ❌ Нет файла `vercel.json`
- ❌ Нет папки `.vercel`
- ❌ В `package.json` нет команд для Vercel

## Рекомендации

### Если хотите использовать только REG.Ru:

1. **Проверьте DNS:**
   - Убедитесь, что A-запись указывает на `31.31.196.49`
   - Не должно быть CNAME на Vercel

2. **Отключите Vercel (если был подключен):**
   - Зайдите на https://vercel.com
   - Найдите проект (если есть)
   - Удалите домен `takto.space` из проекта
   - Или удалите весь проект

3. **Проверьте GitHub Actions (если есть):**
   - Зайдите на https://github.com/ignatovivin/vibesite
   - Перейдите в **"Actions"**
   - Проверьте, нет ли автоматических деплоев на Vercel

### Если хотите использовать Vercel вместо REG.Ru:

1. **Настройте Vercel:**
   - Зайдите на https://vercel.com
   - Подключите репозиторий `ignatovivin/vibesite`
   - Настройте домен `takto.space`

2. **Обновите DNS:**
   - В панели REG.Ru измените A-запись на IP Vercel
   - Или используйте CNAME на `cname.vercel-dns.com`

3. **Удалите FTP деплой:**
   - Можно оставить скрипт, но не использовать его

## Проверка текущего состояния

Выполните команду для проверки DNS:

```bash
# Windows PowerShell
nslookup takto.space

# Linux/Mac
dig takto.space
```

Или используйте онлайн-сервис:
- https://dnschecker.org/
- https://www.whatsmydns.net/

**Ожидаемый результат:**
- A-запись должна указывать на `31.31.196.49` (REG.Ru)

## Если есть проблемы

Если сайт не открывается или показывает старую версию:

1. **Проверьте DNS** (см. выше)
2. **Проверьте, нет ли деплоя на Vercel** (зайдите на vercel.com)
3. **Очистите кэш браузера** (Ctrl+F5)
4. **Подождите 5-15 минут** (распространение DNS/кэша)
